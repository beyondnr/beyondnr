### 2) 가치 제안 통합 분석

- 고객 핵심 문제(Pain)
  - 복잡성 장벽: 기존 SaaS ‘기능 과잉’ → 학습/사용 불안(사회적 수행 불안)로 도입 포기
  - 숨은 리스크: 엑셀/수기 기반에서 주휴·연장/야간/휴일수당·주52시간 등 계산 오류 → 과태료/체불 리스크
  - 커뮤니케이션 혼란: 카톡/전화 의존 변경·대타 조율 → 누락·오류·과부하

- 고객의 목표(JTBD)
  - 법을 준수하는 안전한 급여·스케줄 운영(불안 0, 오류 0, 증빙 확보)
  - 사장은 승인만: 직원 가용 시간 제출 → 자동 취합/검증 → 1클릭 공지
  - ‘엑셀보다 쉬운’ 초간단 사용성으로 빠른 온보딩

- 우리 솔루션의 핵심 제안(Value Proposition)
  - 극단적 단순성(Less is More): ‘엑셀보다 쉬운’ 흐름 중심 UI, 사장님 기준 3단계 온보딩(가입→매장→첫 스케줄)
  - 자동화 리스크 관리: 주휴/연장/야간/휴일수당 자동 계산, 주52시간·휴게·연속휴식 위반 실시간 경고
  - 승인 기반 운영: 직원 입력 → 사장 1클릭 승인·공지 → 출퇴근 기록 연동
  - 신뢰 가능한 증빙: 변경 이력(Audit Log), 법규 기준 계산 근거 보관·내보내기(PDF/엑셀)

#### 2-1) 우리가 제공하는 차별적 가치 (Differentiators)
- 사장님 기준 UX: ‘기능 축소’로 학습 제거, 3단계 온보딩, 오프라인 친화 출력
- 규칙 엔진 기반 법규 준수: 기본 규칙셋 내장 + 변경 시 테이블 업데이트
- 기록의 신뢰성: 변경 이력/근거 자동 축적 → 분쟁 시 증빙
- 대리인 모드: 자녀/직원/세무사가 대신 초기 설정(도입 장벽 제거)

#### 2-2) 기존 대안(Competitor/Substitute)과 격차
- 엑셀/수기: 쉬워 보이나 법규 계산·증빙·변경 이력에 취약
- 카톡/캘린더: 커뮤니케이션 단편화, 스케줄·급여·법규 일체화 불가
- 기존 SaaS(알밤/시프티 등): 고기능이나 학습 난이도 높아 Q4 도입 실패 빈발
- 세무사/노무사: 스케줄/근태/실시간 운영은 대행하지 않음

#### 2-3) Proof (데이터 근거)
- 50대 이상 소상공인 비중 54.8%, 디지털 전환 실패 1위 ‘활용 능력 부족’ → ‘극단적 단순성’ 필요
- 임금체불액 2024 상반기 1조 원 돌파, 원인: 복잡한 노동법규 무지 → ‘자동화 리스크 관리’ 필요
- Q4(핵심 타겟) 명시, 1년 차 SOM 목표 1% ≈ 1,900개 사업체, 연매출 ~4.56억(월 2만 원) 가설

### 4) 가치 제안 선언문 (One-liner)

“엑셀보다 더 쉬운 1클릭 승인과 자동 법규 계산으로, 소상공인 사장님이 불안 없이 ‘법대로’ 스케줄·급여를 운영하게 합니다.”

---

### 5) 성공 지표(Activation/Value) 가설

- 온보딩: 회원가입→매장→첫 스케줄 생성 10분 이내 완료율 ≥ 70%
- 운영: 급여/수당 계산 오류 0건 유지율 ≥ 95%, 위반 사전 경고 처리율 ≥ 80%
- 시간 절감: 월말 정산 소요시간 90% 단축(셀프 리포트+이벤트 로그)
- 신뢰성: 증빙 내보내기 활용률 ≥ 50%, 분쟁/클레임 건수 감소 추세

---

### 6) 초기 GTM · 온보딩 전략 (Q4 집중)

- 채널: 세무사/노무사 파트너십(리스크 경고 맥락), 상권 오프라인(전단/현수막/QR), 동종 사장님 커뮤니티
- 메시지: “엑셀보다 쉬움 · 법대로 자동 계산 · 사장님은 승인만”
- 온보딩: 대리인 모드(자녀/직원 초기 설정), 템플릿(주간/야간 교대 규칙) 제공, 전화 1회 컨시어지
