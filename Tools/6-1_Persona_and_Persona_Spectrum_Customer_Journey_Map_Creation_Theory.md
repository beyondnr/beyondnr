# 페르소나와 페르소나 스펙트럼, 고객 여정지도 작성하기

## 1. 페르소나 (Persona)

AI 시대의 페르소나는 더 이상 정적 프로필이 아니라, **"행동과 맥락의 시뮬레이션 모델"**이 되어야 합니다.

| 구분                    | 전통적 페르소나 (사용안함)             | AI 확장형 페르소나 (채택)                       |
| :---------------------- | :------------------------------------- | :---------------------------------------------- |
| **초점**          | 인구통계, 직업, 연령 등 정적 속성 중심 | 과업·문제·맥락 등 동적 상황 중심              |
| **데이터 소스**   | 설문/관찰 중심                         | AI 기반 데이터 리서치, 소셜 트렌드, JTBD 프리뷰 |
| **활용 목적**     | 대표 사용자 이미지화                   | 시나리오 시뮬레이션, 인터뷰 대상 선정           |
|                                        |                                                 |

### AI 프롬프트 예시

* **잘된 예:** "내 비즈니스는 [서비스명]이야. (참고자료 첨부) 이 서비스를 위한 TAM-SAM-SOM + Market Segment를 참고하여... Market Segment가 정의하고 있는 고객 그룹 유형들로부터 페르소나를 추출하고..."

## 2. 페르소나 스펙트럼 (Persona Spectrum)

극단적·비전형적 사용자까지 포함해 서비스의 포용성(Inclusivity)과 사용 맥락의 다양성을 확보하는 분석 방법입니다.

### 스펙트럼 유형

1. **핵심(Core):** 이상적인 주요 고객군. 매출/사용 빈도 중심.
2. **확장(Adjacent):** 유사 문제를 가진 인접 시장군. 유사 니즈/다른 맥락.
3. **극단(Extreme):** 제약·극단적 상황을 가진 사용자. 실패·불편·대안 사용 중심.
4. **비활성(Non-user):** 아직 사용하지 않거나 회피하는 집단. 인식·가치·신뢰 결여 중심.

### AI 기반 다중 페르소나 생성 워크플로우

1. **1차 생성:** 각 스펙트럼 유형별 3~5개 버전 생성 (Divergent Thinking)
2. **2차 평가:** 문제·행동·맥락의 현실성 기준으로 평가 및 선별 (Convergent Filtering)
3. **3차 보정:** 사업 주제에 맞게 언어 및 상황 맞춤화 (Contextual Rewriting)
4. **4차 검증:** 다중 AI 교차검증 (AI Peer Review)
5. **5차 통합:** 관계 구조화 및 시각화 (Spectrum Map)

### 페르소나 평가 기준

* **현실성:** 실제 시장에서 존재할 확률은? (데이터 근거)
* **차별성:** 다른 페르소나와 뚜렷하게 다른가? (행동·맥락)
* **통찰성:** Pain 정의가 유효한가? (진보 욕구 중심)
* **전략성:** MVP 설계에 인사이트를 주는가? (기능·메시지 매핑)

## 3. 고객 여정지도 (Customer Journey Map, CJM)

고객이 "문제 인식 → 해결 탐색 → 구매/사용 → 평가/유지" 과정에서 겪는 경험, 감정, Pain Point를 시각화한 흐름도입니다.

### 고객 여정의 단계

인지(Awareness), 고려(Consideration), 결정(Decision), 온보딩(Onboarding), 충성도(Loyalty)

### CJM 예시 1: 가구 시장 ("실측" & "조립/설치")

* **탐색:** 공간 활용 방법 탐색 (유튜브/네이버) → "따라하기만 해도 되겠다"
* **실측:** 가구 공간 결정 및 사이즈 실측 → Pain: "줄자가 없는데...", "사이즈로 검색이 안되네"
* **검색/비교:** 사이즈에 맞는 가구 검색, 상세 페이지 확인 → Pain: "치수 표시 안됨", "하나하나 비교해야 하나?"
* **구매/배송:** 구매 진행, 상품 배송 → "결제수단 다양", "배송 빠름"
* **조립/설치:** 가구 조립 → "한국어 매뉴얼 자세함"
* **리뷰 작성:** 리뷰 작성 → "사은품 좋음"

### CJM 예시 2: 자동차 시장 (Emotional Eric)

* Consider, Explore, Compare, Test, Negotiate 단계를 거치며 웹사이트, 앱, 딜러십 방문 등의 터치포인트에서 발생하는 긍정/부정 경험을 매핑.

### CJM 작성 구조

| 단계                         | 고객 행동             | 고객 생각                    | 감정           | Pain Point             | 개선 기회               |
| :--------------------------- | :-------------------- | :--------------------------- | :------------- | :--------------------- | :---------------------- |
| **문제 인식**          | 불편함을 느낌         | "이 문제 왜 생기지?"         | 불안           | 정보 부족              | 문제 원인 콘텐츠 제공   |
| **탐색/의사결정/사용** | 대안 탐색, 선택, 사용 | "괜찮아 보이는데?", "어렵네" | 기대/불안/실망 | 복잡한 비교, 신뢰 부족 | 비교 툴, 후기/데모 제공 |
| **유지**               | 반복 사용             | "익숙해졌네"                 | 안정           | 가치 감소              | 리텐션 캠페인           |

### AI 생성 프롬프트 예시

"내 서비스의 주요 고객 페르소나를 [타깃]으로 두고, 그가 내 서비스를 접하기 전부터 구매 후까지 겪는 주요 행동/생각/감정을 고객 여정지도 형태로 정리해줘. 단계별 정확하고 간결하게 정의된 'Pain Point' 서술이 반드시 포함되어 있어야 해."
